# Toolkit Scratch Pad Protocol

## Purpose
Central scratch file for complex strings that fail JSON parameter escaping in LM Studio plugin tools.

## Configuration
- Config file: `.toolkit-config.json`
- Scratch path: `C:\Users\UserMN4312\toolkit\lm-studio-plugin\.scratch-pad.txt`
- Env override: `TOOLKIT_SCRATCH_PAD`

## Usage Pattern

### Step 1: Write Complex Content
```
write_file(path="C:\Users\UserMN4312\toolkit\lm-studio-plugin\.scratch-pad.txt", content="complex regex pattern here")
```

### Step 2: Process via run_command
```
run_command(command="rg -f C:\\Users\\UserMN4312\\toolkit\\lm-studio-plugin\\.scratch-pad.txt <directory>")
```

### Step 3: Clear When Done
```
write_file(path="C:\Users\UserMN4312\toolkit\lm-studio-plugin\.scratch-pad.txt", content="")
```

## Tool Fallback Map

| Tool | When to Use Scratch | Fallback Command |
|------|-------------------|------------------|
| `grep` | Complex regex patterns | `rg -f <scratch> <dir>` |
| `edit_file` | Multi-line deletes | `powershell -c (Get-Content...)` |
| `cat_multiple` | Non-JSON files | `cat` each file individually |
| `create_file` (JSON) | Complex JSON | `write_file` with manual formatting |
| `search_directory` | Large result sets | `run_command` with `dir /s /b` |

## Escape Issues Catalog

### Issue 1: Regex Patterns
**Symptom:** `grep` tool fails with complex regex
**Cause:** Special characters (`\s`, `\w`, `+`, `*`) get double-escaped
**Fix:** Write pattern to scratch, use `rg -f`

### Issue 2: Backslashes in Paths
**Symptom:** File paths with `\` break tool parameters
**Cause:** JSON serialization escapes `\` to `\\`
**Fix:** Use forward slashes `/` or write path to scratch

### Issue 3: Nested Quotes
**Symptom:** Strings with `'` inside `"..."` fail
**Cause:** Quote escaping conflicts between JSON and shell
**Fix:** Write to scratch, reference via file path

### Issue 4: Multi-line Content
**Symptom:** Multi-line strings break in tool parameters
**Cause:** Newlines must be escaped as `\n` in JSON
**Fix:** Write to scratch with actual newlines

## Discovery

### How Tools Find the Config
1. Check `TOOLKIT_SCRATCH_PAD` env var (override)
2. Read `.toolkit-config.json` in plugin root
3. Default to `.scratch-pad.txt` in plugin root

### How Humans Find It
- Config: `.toolkit-config.json` in plugin root
- Protocol: `.toolkit-scratch-protocol.md` in plugin root
- Scratch: `.scratch-pad.txt` in plugin root
